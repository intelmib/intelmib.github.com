# Politica de Privacidade - Protetor de Chamadas

Ultima atualizacao: 2026-04-30

Esta Politica de Privacidade descreve como o app **Protetor de Chamadas** (`com.intelmib.callblocker`) acessa, usa, armazena e protege dados no dispositivo do usuario.

## Desenvolvedor e contato

Desenvolvedor: Intelmib  
Contato de privacidade: renannhola@gmail.com

## Dados acessados pelo app

O app usa permissoes relacionadas a telefone para executar sua funcao principal de identificar chamadas e aplicar regras de bloqueio configuradas pelo usuario.

O app pode acessar os seguintes dados no dispositivo:

- Estado do telefone, para detectar chamadas recebidas e acionar o fluxo de bloqueio.
- Numero de telefone de chamadas recebidas, quando disponibilizado pelo Android.
- Historico de chamadas do dispositivo, quando o usuario concede permissao.
- Regras de bloqueio criadas pelo usuario, como numeros, padroes e rotulos.
- Registros locais de chamadas analisadas e chamadas bloqueadas.

## Finalidade do uso dos dados

Os dados sao usados somente para:

- Aplicar regras de bloqueio de chamadas.
- Exibir historico de chamadas e chamadas bloqueadas dentro do app.
- Permitir que o usuario crie, remova e consulte regras de bloqueio.
- Melhorar a experiencia local do usuario no proprio dispositivo.

## Compartilhamento de dados

O app nao envia, vende ou compartilha dados pessoais com terceiros.

Os dados de chamadas e regras sao processados localmente no dispositivo. O app nao possui conta de usuario, login, servidor proprio ou sincronizacao externa.

## Armazenamento e seguranca

Os dados necessarios ao funcionamento do app sao armazenados localmente no armazenamento privado do aplicativo no Android.

O acesso a esses arquivos e limitado pelo modelo de seguranca do Android. Mesmo assim, o usuario deve manter o dispositivo protegido com senha, PIN, biometria ou outro mecanismo de seguranca.

## Retencao e exclusao de dados

Os dados permanecem no dispositivo enquanto o app estiver instalado ou ate que o usuario os remova.

O usuario pode excluir registros dentro do app quando houver opcao disponivel, remover regras individualmente ou apagar todos os dados do app pelas configuracoes do Android. Desinstalar o app tambem remove os dados locais associados ao aplicativo.

## Permissoes sensiveis

O app pode solicitar permissoes como `READ_PHONE_STATE`, `READ_CALL_LOG` e `ANSWER_PHONE_CALLS` para executar sua funcionalidade principal de protecao contra chamadas indesejadas.

Se o usuario negar permissoes necessarias, algumas funcoes podem nao operar corretamente.

## Alteracoes nesta politica

Esta politica pode ser atualizada para refletir mudancas no app, nas permissoes usadas ou nos requisitos legais e de plataforma.

Quando houver mudancas relevantes, a data de atualizacao sera alterada nesta pagina.
